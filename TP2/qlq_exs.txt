src/app :
	Ce dossier contient tout le code TypeScript, HTML et CSS de votre application :
	les composants (.component.ts, .html, .css),les services,les modules,et la logique principale du projet.
src/assets :
	Contient les ressources statiques (images, icônes, fichiers JSON, etc.)
	Ces fichiers ne sont pas modifiés par Angular et sont copiés tels quels lors de la compilation.
src/environments :
	Contient les fichiers de configuration par environnement :
		environment.ts -> configuration pour le développement
		environment.prod.ts -> configuration pour la production.
src/main.ts:
	C'est le point d'entrée principal de l'application Angular.
	Il démarre l'application en chargeant le module racine (AppModule) dans le navigateur.
src/index.html:
	Le fichier HTML principal de l’application.Il contient la structure de base du site et la balise racine.
angular.json:
	Le fichier de configuration du projet Angular.
	Il décrit :
		les chemins des fichiers sources
		les scripts et styles à inclure
		les options de compilation
		les paramètres de build et de déploiement
package.json:
	Liste toutes les dépendances du projet et les scripts npm pour exécuter ou compiler l’application :
		"dependencies" : bibliothèques nécessaires à l’exécution,
		"devDependencies" : outils nécessaires au développement,
		"scripts" : commandes (ex. ng serve, ng build…).
tsconfig.json:
	Fichier de configuration TypeScript.
	Il définit les règles de compilation du code TypeScript vers JavaScript :
		la version cible de JavaScript
		les répertoires inclus/exclus
		les options de vérification du code.
